## 🧠 Key Concepts Practiced

- **User Stories** — As a [who], I want [what], so that [why]
- **Story Points** — Estimating complexity (1, 2, 3, 5, 8)
- **Sprint Planning** — Splitting work into weekly cycles with velocity caps
- **Definition of Done** — Clear acceptance criteria per story
- **BLUF Standup** — Bottom Line Up Front: completed → detail → today → blocker
- **Risk Register** — Likelihood, Impact, Mitigation for each risk
- **Scope Change Handling** — Negotiate, Defer, Spike, or Accept

---

## 💡 Key Takeaways

- Agile does not add more work — it adds **clarity** about what the work actually is
- A standup is not a status report — it is a **coordination tool** (90 seconds, BLUF)
- Story points are not deadlines — they are **conversations about complexity**
- The right answer to scope change is not yes or no — it is **"yes, and here is the trade-off"**
- Naming a blocker is not weakness — **hiding it is**
